🔑 RSA (Rivest-Shamir-Adleman)

RSA is an asymmetric encryption algorithm based on the mathematical difficulty of factoring large prime numbers. It uses a public key for encryption and a private key for decryption.

How it works: Choose two prime numbers p and q. Compute n = p × q and φ(n) = (p-1)(q-1). Choose e coprime to φ(n), then compute d = e⁻¹ mod φ(n). Public key is (e, n), private key is (d, n).

📈 Elliptic Curve Cryptography (ECC)

ECC is based on the algebraic structure of elliptic curves over finite fields. It offers the same level of security as RSA with much smaller key sizes.

Curve equation: y² = x³ + ax + b (mod p)
Key idea: Given a base point G and a scalar k, computing Q = kG is easy, but finding k from Q and G (the Elliptic Curve Discrete Log Problem) is extremely hard.

🤝 Diffie-Hellman Key Exchange (DHKE)

DHKE allows two parties to establish a shared secret over an insecure channel without prior shared secrets. It's based on the difficulty of the Discrete Logarithm Problem.

Protocol:
1. Agree on public parameters: prime p and generator g
2. Alice picks secret a, sends A = g^a mod p
3. Bob picks secret b, sends B = g^b mod p
4. Shared secret: Alice computes B^a mod p, Bob computes A^b mod p
Both get: g^(ab) mod p — the same shared secret!

🕵️ Eve's Perspective (Eavesdropper)

Eve can see the public values: p, g, A = g^a mod p, and B = g^b mod p.
But to find the shared secret, she would need to solve the Discrete Logarithm Problem — finding a from g^a mod p — which is computationally infeasible for large primes!
